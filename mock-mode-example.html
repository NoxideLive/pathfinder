This guide demonstrates how to enable and use the mock/dev mode for Pathfinder UI development. Mock mode allows you to develop and test the UI without requiring a live backend server.

‚ö†Ô∏è Important: Mock mode is for development only. Never use in production!
üîí Production Safety: Mock mode is controlled by MOCK_ALLOWED in app/environment.ini. It is disabled by default in PRODUCTION mode and enabled in DEVELOP mode. Even with URL parameters or localStorage, mock mode will not activate if MOCK_ALLOWED = 0.

Quick Start

Method 1: URL Parameter (Recommended)

Simply add ?mockMode=true to your URL:

http://localhost/pathfinder/?mockMode=true
Try It Now

Method 2: Browser localStorage

Open your browser's console and run:

localStorage.setItem('pathfinder_mock_mode', 'true'); location.reload();

Method 3: Global Configuration

Add this script before loading Pathfinder:

<script> window.PATHFINDER_MOCK_MODE = true; </script>

Available Mock Data

The following API endpoints are currently mocked:

  • /api/Map/initData - Application initialization
  • /api/User/getEveServerStatus - Server status
  • /api/User/getCookieCharacter - Character data
  • /api/Map/updateUserData - User data updates
  • /api/Map/updateData - Map data updates

Advanced Configuration

You can configure mock mode behavior in the browser console:

// Disable delay simulation MockInterceptor.configure({ simulateDelay: false }); // Change delay range (100-1000ms) MockInterceptor.configure({ delayMin: 100, delayMax: 1000 }); // Simulate 10% failure rate MockInterceptor.configure({ failureRate: 0.1 }); // Disable console logging MockInterceptor.configure({ logRequests: false });
